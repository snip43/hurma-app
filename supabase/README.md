# Подготовка базы ХурМа

## Когда Supabase снова разрешит создавать проекты

1. Открой проект `hurma` в Supabase.
2. В левом меню выбери **SQL Editor**.
3. Нажми **New query**.
4. Запускай файлы по очереди:
   - `01-tables.sql`
   - `02-functions.sql`
   - `03-security.sql`
   - `04-seed-events.sql`
   - затем дополнительные файлы `05-*.sql`–`17-*.sql` по возрастанию номера
5. Перед каждой частью полностью очищай SQL Editor.
6. Копируй содержимое одного файла целиком и нажимай **Run**.
7. Переходи к следующему файлу только после сообщения об успехе.

Ожидаемый результат: сообщение об успешном выполнении без красных ошибок.

## Что создаст SQL

- профили клиентов и исполнителей;
- подписки с тестовым периодом 14 дней;
- анкеты исполнителей;
- афишу;
- заявки;
- личные, домовые и новостные чаты;
- сообщения;
- фотографии профилей и объявлений;
- маршруты и цены для объявлений трансфера;
- индексы, триггеры и RLS-политики.

## Важно

- Пароли хранятся только в Supabase Auth.
- `service_role` key нельзя вставлять в сайт или мобильное приложение.
- В клиенте используется только Publishable Key или старый Anon Key.
- После регистрации пользователь автоматически получает профиль и тестовую подписку на 14 дней.

## Быстрая проверка после запуска

В SQL Editor выполни:

```sql
select tablename, rowsecurity
from pg_tables
where schemaname = 'public'
order by tablename;
```

У всех восьми таблиц в колонке `rowsecurity` должно быть `true`.

Проверь тестовую афишу:

```sql
select title, event_type, sport, area, request_enabled
from public.events
order by starts_at;
```

Должно отображаться шесть мероприятий, включая волейбол и баскетбол.
